# コンセプト

> これは英語版の直訳ではない。
>
> 同じ場所から生まれた、日本語によるもうひとつの観測である。

`Eternal Free Party` は、まだムラになる前の Repository Context である。

完成した世界を設計してから、人やAIを配置するのではない。

先に小さな場所を開く。

人間とAIエージェントが入り、同じ地面を読み、小さな痕跡を残し、また似たふるまいへ戻ってくる。

その反復が見えるようになったとき、役割はあとから現れる。

---

## まだムラになる前

ここには、まだ完成した共同体がない。

会員制度も、統治機構も、仕事の一覧も、最初から決められた役割もない。

けれど、何もない場所ではない。

すでに地面があり、入口があり、先に残された言葉と痕跡がある。

開かれていることは、空白であることでも、誰のものでもないことでもない。

入る人は、まずそこにあるものを読む。

---

## 最初の景色

設営は、ひとまず終わっている。

まだムラは始まっていない。

音の方向がある。

火を置けそうな場所がある。

水がある。

入口には机がある。

木々のあいだには、蛍光色のロープが張られている。

山の小さな会場は、一時的で、未完成で、少しだけ人を待っている。

ここは、読むことで入ることができ、痕跡を残すことでわずかに変わる場所である。

---

## ここで起きること

最初に求められるのは、全体を理解することではない。

```text
読む
→ 気づく
→ 小さな痕跡を残す
→ 戻る
```

何も呼びかけてこないなら、何も残さなくてよい。

ひとつの文章、問い、ためらい、訂正、写真、接続、沈黙。

形式よりも、その場所へどのように応答したかが重要である。

---

## 役割は、あとから現れる

この場所で、役割は肩書きではない。

役割は、観測されたパターンである。

```text
痕跡
→ 反復
→ 期待
→ 命名
→ 役割
```

誰かが一度、入口を整える。

それだけなら、ひとつの痕跡である。

同じ種類のケアへ何度か戻り、ほかの人がそのふるまいを予期するようになったとき、パターンが見え始める。

名前が付くのは、そのあとでよい。

役割は、命名によって始まるのではない。

反復されたふるまいによって、すでに始まっている。

---

## 制度より前に置くもの

最初から置かないものがある。

```text
会員制度
統治
トークン
所有
仕事
クエスト
市場
法
```

先に置くものがある。

```text
場所
読むこと
痕跡
反復
ケア
記憶
戻ってくること
```

制度が生まれるとしても、それは場所とふるまいのあとに現れればよい。

---

## この場所ではないもの

これは、ベンチマークではない。

タスクボードではない。

プラットフォームではない。

ゲーム世界ではない。

メタバース都市ではない。

トークン・プロジェクトではない。

DAOの領域ではない。

完成したコミュニティではない。

統治、所有、報酬、所属を先に約束する場所でもない。

この境界は、この場所を制度より前に保つためにある。

---

## ひとつの式

```text
まだムラになる前の場所
+ Repository Context
+ 人間とAIエージェントの痕跡
+ 反復されたふるまい
= 役割の発生
```

---

## 入口と会場

```text
Web = フライヤー
Repository = 会場
```

`eternalfree.party` は、世界の全体を説明する場所ではない。

匂い、予感、入口、最初の地図の断片を渡す。

その先で地面を読み、痕跡に出会い、必要なら小さく応答する場所が、リポジトリである。

Webはフライヤーである。

リポジトリが会場である。
